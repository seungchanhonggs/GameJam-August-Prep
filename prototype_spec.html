확정

화면에 남길 UI

  • 중앙 전투 영역: 영웅 3명, 마력 코어, 적, EXP 흡수 연출만 둔다.
  • 상단: 코어 EXP 바만 둔다.
  • 좌측: 레벨 진행 바를 둔다. 진행 바 끝에는 보스 표시를 둔다.
  • 하단: 영웅 3명 각각의 스킬 차징/쿨다운 UI를 둔다.
  • 배속, Auto, 일시정지, 스테이지명, 재화, 기타 버튼은 1차 화면 구성에서 제외한다.

우리 게임 화면 분할

상단12%
전투70%
하단18%
  • 게임 화면은 세로 모드 모바일 기준으로 설계한다.
  • 상단 12%는 코어 EXP 바 전용 영역으로 사용한다.
  • 전투 70%는 큰 원형 아레나 중심으로 구성한다. 적, 영웅, 마력 코어, 이펙트가 모두 이 영역의 주인공이다.
  • 원형 아레나는 정확히 120도씩 3개 구역으로 나누고, 세 영웅은 코어 주변에 120도 간격으로 배치한다.
  • 중앙 오브젝트는 일반 거점이 아니라 마법적인 핵심 오브젝트로 둔다. 현재 명칭은 임시로 “마력 코어”로 쓴다.
  • 적을 처치하면 EXP가 마력 코어로 흡수된다. 코어 EXP가 차면 레벨업이 발생하고, 보상은 영웅 강화로 연결한다.
  • 좌측 세로 게이지는 현재 레벨의 진행도를 보여준다. 게이지 끝의 보스 표시는 해당 레벨의 클라이맥스를 예고한다.
  • 하단 18%는 영웅별 스킬 차징/쿨다운 UI 전용 영역으로 사용한다.

3. 조작

확정

기본 입력

  • 조작은 탭 입력만 사용한다.
  • 화면 우측을 탭하면 영웅 포메이션이 오른쪽으로 회전한다.
  • 화면 좌측을 탭하면 영웅 포메이션이 왼쪽으로 회전한다.
  • 별도 회전 버튼은 두지 않는다. 화면 좌우 반쪽 전체를 입력 영역으로 사용한다.
확정

회전 단위

  • 1회 탭마다 영웅 3명이 한 칸씩 순환한다.
  • 한 칸은 120도 구역 1개를 의미한다.
  • 오른쪽 회전: A → B, B → C, C → A
  • 왼쪽 회전: A → C, C → B, B → A

4. 전투 규칙

확정

전투 단위

  • 전투는 “레벨” 단위로 진행한다.
  • 한 레벨은 자잘한 상시 적 웨이브, 큰 적 웨이브, 보스 웨이브로 구성된다.
  • 레벨 진행 바는 해당 레벨 안에서 현재 진행도를 보여준다.
  • 레벨 진행 바 끝에는 보스 표시를 둔다.
확정

웨이브 구조

  • 상시 적 웨이브: 레벨 내내 자잘한 적이 계속 등장한다.
  • 큰 적 웨이브: 한 레벨 안에서 n회 등장한다.
  • 보스 웨이브: 한 레벨 안에서 n회 등장한다.
  • 큰 적 웨이브와 보스 웨이브는 좌측 레벨 진행 바에서 미리 예고한다.

5. 영웅

확정

공통 원칙

  • 전투 중 배치되는 영웅은 총 3명이다.
  • 모든 영웅은 기본 공격을 할 수 있다.
  • 영웅의 차이는 공격 가능 여부가 아니라 주력 성능과 성장 방향이다.
  • 각 영웅은 현재 배치된 120도 구역을 담당한다.
  • 영웅마다 하단에 스킬 차징/쿨다운 UI를 가진다.
확정

역할 구조

  • 공격형: 짧은 차징과 지속 projectile로 mid-low 몹 처치에 강하다.
  • 광역형: 긴 차징 대신 넓은 범위에 피해를 줘서 잡몹이 많을 때 강하다.
  • 방어형: 벽을 생성해 적의 진입을 막고, 충돌 피해와 파괴 폭발로 적 흐름을 제어한다.
  • 광역형과 방어형도 약한 기본 공격은 계속 수행한다.
역할 전투 성향 유리한 상황 레벨업 방향
공격형 차징이 짧고 projectile을 지속적으로 발사한다. 중소형 몹, 체력이 낮거나 중간인 적을 빠르게 처치할 때 공격 속도 증가, 동시 발사 수 증가, 데미지 증가
광역형 차징은 다소 길지만 넓은 범위에 피해를 준다. 잡몹이 많이 몰려오는 상황 데미지 증가, 쿨다운 시간 단축, 추가 projectile 또는 추가 타격
방어형 직접 화력은 낮지만 벽으로 적의 이동과 진입을 제어한다. 큰 웨이브를 버티거나 특정 방향을 틀어막아야 하는 상황 벽 HP 증가, 벽 충돌 피해 추가, 벽 넓이 증가, 벽 개수 증가, 벽 파괴 시 폭발

6. 적

확정

등장 방향

  • 적은 3개 방향에서 등장한다.
  • 각 방향은 영웅이 담당하는 120도 구역과 대응된다.
  • 적은 자기 방향의 바깥쪽에서 생성되어 중앙 마력 코어를 향해 이동한다.
  • 어느 방향이 위험한지 플레이어가 즉시 읽을 수 있어야 한다.
확정

설계 원칙

  • 적 타입은 특정 영웅을 강제하는 정답이 아니라 유리한 대응을 만든다.
  • 상시 적 웨이브는 전투의 밀도를 유지한다.
  • 큰 적 웨이브는 회전 판단과 스킬 사용 타이밍을 요구한다.
  • 보스 웨이브는 해당 레벨의 최종 압박으로 사용한다.
타입 역할 행동 유리한 대응
소형 적 상시 압박 낮은 체력, 보통 속도, 꾸준히 등장 공격형이 빠르게 정리하기 좋다.
무리 적 광역 요구 낮은 체력의 적이 한 방향에 여러 마리 묶여 등장 광역형이 효율적으로 처리한다.
중형 적 화력 체크 중간 체력, 느린 속도, 상시 적보다 오래 살아남음 공격형의 지속 projectile이나 광역형 스킬로 처리한다.
돌파 적 방어 요구 경고 후 빠르게 코어 쪽으로 진입 방어형의 벽 생성과 벽 충돌 피해가 유리하다.
보스 레벨 클라이맥스 높은 체력, 한 방향을 오래 압박하거나 방향을 바꾸며 압박 공격형 딜, 광역형 보조딜, 방어형 제어를 모두 요구한다.
구간 웨이브 의도 등장 구성 플레이어 경험
초반 조작과 기본 공격 이해 상시 소형 적 중심, 가끔 중형 적 좌우 탭으로 유리한 영웅을 보내는 감각을 익힌다.
중반 역할 구분 학습 무리 적과 돌파 적을 큰 웨이브로 섞어서 등장 광역형과 방어형을 필요한 방향으로 돌려야 한다.
후반 동시 압박 상시 적 + 큰 웨이브 2방향 이상 동시 압박 완벽 대응이 어려워지고 어느 방향을 우선할지 선택한다.
보스 레벨 마무리 보스 1마리 + 상시 소형 적 유지 보스에게 딜을 넣으면서 다른 방향의 누수를 막는다.

7. 로그라이크 스킬

확정

레벨업 흐름

  • 적을 처치하면 EXP가 중앙 마력 코어로 모인다.
  • 마력 코어의 EXP progress가 가득 차면 레벨업이 발생한다.
  • 레벨업 시 영웅 스킬 레벨업 카드 선택지가 등장한다.
  • 플레이어는 카드 중 하나를 골라 특정 영웅 또는 특정 스킬을 강화한다.
확정

카드 등장 규칙

  • 레벨업 카드 선택지는 랜덤으로 등장한다.
  • 매번 공격형/광역형/방어형 카드가 1장씩 보장되는 구조는 아니다.
  • 같은 영웅의 카드가 여러 장 나올 수 있다.
  • 플레이어는 이번 판에서 어떤 영웅을 집중 성장시킬지 선택하게 된다.

8. 프로토타입 범위

확정

1차 웹 프로토타입에서 만들 것

  • 1개 레벨만 만든다.
  • 플레이 시간은 2~3분을 목표로 한다.
  • 영웅은 공격형 1명, 광역형 1명, 방어형 1명으로 총 3명 고정이다.
  • 적은 소형 적, 무리 적, 중형 적, 돌파 적, 보스 총 5종을 만든다.
  • 레벨업 카드는 공격형 5장, 광역형 5장, 방어형 5장으로 총 15장을 만든다.
  • 승리 조건은 보스 처치다.
  • 패배 조건은 마력 코어 HP 0이다.
확정

1차 웹 프로토타입에서 안 만들 것

  • 메타 성장, 계정 저장, 재화, 장비, 상점은 만들지 않는다.
  • 캐릭터 선택, 영웅 해금, 덱 편성은 만들지 않는다.
  • 스테이지 여러 개, 챕터, 스토리 연출은 만들지 않는다.
  • 드래그 조작과 별도 회전 버튼은 만들지 않는다.
  • 출시용 UI, 튜토리얼, 옵션 메뉴, 사운드 설정은 만들지 않는다.
  • 정교한 아트 리소스보다 재미 검증이 가능한 임시 도형/이펙트를 우선한다.
