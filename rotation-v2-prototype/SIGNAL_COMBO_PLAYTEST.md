# 신호갑주 트리코어 플레이테스트

## 이번 빌드의 가설

같은 적을 방어형 → 광역형 → 궁수형에게 차례로 넘기는 과정이 현재 회전 조작의 핵심 재미가 될 수 있다.

## 일반 플레이에서 확인하기

1. `rotation-v2-prototype`을 실행한다.
2. 자유 회전 또는 120도 스위치 모드를 선택한다.
3. 전투를 시작한다.
4. 약 12초에 첫 신호갑주가 등장한다.
5. 체력을 60%까지 낮추면 삼색 신호가 열린다.
6. 초록 방어형 → 노랑 광역형 → 빨강 궁수형 순서로 공격 범위를 맞춘다.

## 빠른 디버그 확인

디버그 주소: `http://127.0.0.1:5176/rotation-v2-prototype/?debug`

- `S`: 신호 노출 직전 체력의 트리코어를 가까운 위치에 생성
- `Q`: 테스트용 120도 좌회전
- `E`: 테스트용 120도 우회전
- `M`: 기존 중형 적 3방향 생성
- `L`: 코어 영웅 공격 완충

일반 전투에서는 약 12초에 1마리가 먼저 등장하고, 이후 34·64·88·118초에 기본 2마리씩 등장한다. 반복 웨이브 수는 밸런스 패널의 `반복 신호갑주 수`에서 조절한다.

`Q`와 `E`는 두 회전 모드에서 조합 단계를 빠르게 재현하기 위한 디버그 전용 입력이다. 실제 플레이 감각은 화면 좌우 입력으로 별도 확인한다.

## 기대되는 화면 반응

- 체력 60% 이하: `SIGNAL OPEN`, 초록 신호 점멸
- 잘못된 영웅: 피해 감소, `BLOCKED`, 단계 유지
- 방어형 정답: `WEAK`, 적 정지, 노랑 신호 활성화
- 광역형 정답: `WEAK`, 점화, 빨강 신호 활성화
- 궁수형 정답: `CRITICAL!`, `3 HERO COMBO!`, 큰 마무리 피해

## 플레이 후 답할 질문

각 문항을 1점(전혀 아니다)부터 5점(매우 그렇다)까지 기록한다.

1. 설명을 읽지 않아도 현재 필요한 영웅을 알 수 있었다.
2. 적을 세 영웅에게 넘기기 위해 회전하는 과정이 재미있었다.
3. 오답과 정답의 차이가 즉시 느껴졌다.
4. 마지막 궁수 치명타가 충분히 통쾌했다.
5. 같은 조합을 여러 번 다시 해보고 싶었다.

## 구현 게이트

- 2번 또는 5번이 3점 미만이면 나머지 적 3종을 구현하지 않는다.
- 신호 판독이 어렵다면 적 종류를 늘리기 전에 크기·색·아이콘·이펙트를 수정한다.
- 회전이 번거롭다면 제한시간이나 실패 초기화를 추가하지 않고 조작과 공격 타이밍부터 수정한다.
- 재미가 확인된 뒤에만 램혼·블룸백·핀아이와 혼합 웨이브를 구현한다.

## 제작자 기능 검증 기록 — 2026-07-15

- 120도 스위치 모드: 초록 → 노랑 → 빨강 전체 조합 성공
- 자유 회전 판정: 디버그 단계 회전으로 전체 조합 성공
- 일반 전투 시작 약 12초 후 첫 신호갑주, 이후 반복 웨이브가 등장하도록 구성
- `WEAK`, `CRITICAL!`, `BLOCKED`, `3 HERO COMBO!` 표시 확인
- 정답 직후 남은 연사와 방어벽 접촉이 불필요한 `BLOCKED`를 반복하지 않도록 처리
- 밸런스 패널에서 트리코어 체력, 노출 비율, 상성 배수, 정지·점화·마무리 수치 조절 가능
- 브라우저 콘솔 오류 없음

기능 동작과 화면 가독성만 확인했으며, 실제 재미 여부는 사용자 플레이테스트 전까지 미확정이다.
